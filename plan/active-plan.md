### 🟡 Спринт 3: Облачная база данных и Серверные шлюзы (АКТИВНЫЙ СУПЕР-ШАГ)
*Мы переносим проект со стадии локального макета на стадию защищенного облачного бэкенда.*

*   **[ ] Шаг 3.4: Создание API-маршрута записи сделок (`/api/trading/record-trade`)**
    *   *Что пишем:* Серверный Next.js POST-маршрут, который принимает данные о закрытой сделке, проверяет её финансовый итог и вносит запись в базу данных Supabase.
*   **[ ] Шаг 3.5: Создание API-маршрута предохранителя (`/api/trading/circuit-breaker`)**
    *   *Что пишем:* Серверный маршрут проверки риска. Суммирует дневные убытки по БД. При превышении 1% (50 EUR) аппаратно блокирует API открытия ордеров на 24 часа.
*   **[ ] Шаг 3.6: Интеграция API-маршрутов в React-интерфейс**
    *   *Что пишем:* Обновление кода `app-code/app/page.tsx`. Замена локального localStorage на fetch-запросы к API. Настройка подтягивания баланса и блокировки экрана при Circuit Breaker.

### ⚪ Спринт 4: Production HFT & WebSockets (ЗАПЛАНИРОВАН)
*Снижение пинга до <15мс и внедрение Stealth-защиты стопов на сервере во Франкфурте.*

1. **Шаг 4.1:** Развертывание выделенной торговой ноды во Франкфурте рядом с серверами брокеров.
2. **Шаг 4.2:** Переключение на WebSocket-соединение для обработки цен за 5–15 миллисекунд.
3. **Шаг 4.3:** Реализация алгоритма Stealth Stop-Loss (хранение стоп-лоссов в оперативной памяти сервера).
4. **Шаг 4.4:** Настройка серверного EOD Auto-Halt Cron-скрипта (автоматическая остановка торгов в 18:00 CET).

## 🧪 КРИТЕРИИ ВЕРИФИКАЦИИ И DoD (DEFINITION OF DONE)
* **Проверка бэкенда:** Попытка несанкционированного чтения данных другого пользователя через API возвращает ошибку `401 Unauthorized`.
* **Проверка Circuit Breaker:** Искусственный убыток в 50 EUR мгновенно вешает красный блок-экран `PANIC STOP ACTIVE` и блокирует торговые кнопки.
* **Проверка EOD Auto-Halt:** При наступлении 18:00 CET все сделки принудительно закрываются сервером, фиксируя прибыль в сейф.
