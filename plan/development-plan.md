## 🏃‍♂️ SPRINT 3: CLOUD BACKEND & SECURE API [ТЕКУЩИЙ АКТИВНЫЙ СПРИНТ 🟡]
*Главная задача: Построить безопасный серверный мост между React-интерфейсом и СУБД Supabase PostgreSQL.*

### 📍 Шаг 3.1: Настройка облачного коннектора (Выполнено)
*   **Что сделано:** Создан проект Supabase (регион eu-west-3, Париж). Переменные доступа скрыты в `.env.local`. Создан клиент базы данных в `app-code/app/lib/supabase.ts`.

### 📍 Шаг 3.2: Развертывание PostgreSQL схем в облаке (Выполнено)
*   **Что сделано:** В SQL-редакторе Supabase успешно выполнен скрипт создания таблиц `users` (профили и балансы), `trades` (архив сделок) и `system_logs` (EOD и Circuit Breaker логи).

### 📍 Шаг 3.3: Внедрение Row Level Security (RLS) на уровне ядра БД (Выполнено)
*   **Что сделано:** Активирована защита RLS для таблиц. Написаны SQL-политики (`auth.uid() = user_id`), исключающие несанкционированный доступ к данным другого пользователя.

### 📍 Шаг 3.4: Разработка серверного шлюза для записи сделок (В разработке)
*   **Что делаем:** Создаем Next.js API-маршрут `app/api/trading/record-trade/route.ts`.
*   **Как делаем:** Пишем POST-обработчик, который принимает данные о закрытой сделке с клиента, перепроверяет математический финансовый результат на сервере и делает запись в таблицу `trades`.
*   **Зачем:** Прямая запись из браузера запрещена из-за угрозы подделки данных. API-маршрут выступает в роли доверенного посредника.

### 📍 Шаг 3.5: Разработка серверного шлюза Circuit Breaker (В разработке)
*   **Что делаем:** Создаем API-маршрут `app/api/trading/circuit-breaker/route.ts`.
*   **Как делаем:** Пишем GET/POST обработчики, которые на сервере суммируют убытки пользователя за текущие 24 часа. При превышении отметки **1% (50 EUR)** сервер аппаратно блокирует API-эндпоинт открытия новых сделок для данного пользователя на сутки.
*   **Зачем:** Enforce-защита. Проверку риска в браузере легко обойти. Серверный контроль Circuit Breaker обойти невозможно.

### 📍 Шаг 3.6: Интеграция API-маршрутов в React-интерфейс (В разработке)
*   **Что делаем:** Обновляем `app-code/app/page.tsx`, заменяя все обращения к `localStorage` на вызовы `fetch()` к созданным API.
*   **Как делаем:** При загрузке страницы баланс и история подтягиваются напрямую из Supabase. При совершении трейда отправляется запрос на запись. Если сервер возвращает статус "Circuit Breaker Active", интерфейс блокируется красным экраном тревоги.

## 🏃‍♂️ SPRINT 4: PRODUCTION HFT & WEBSOCKETS [ЗАПЛАНИРОВАНО ⚪]
*Главная задача: Снизить пинг исполнения ордеров до <15мс и внедрить Stealth-механизмы защиты от манипуляций.*

*   **📍 Шаг 4.1: Развертывание выделенной торговой ноды во Франкфурте**
    *   *Что сделаем:* Развернем бэкенд на облачном сервере в Франкфурте (AWS/Hetzner) рядом с серверами крупнейших брокеров.
*   **📍 Шаг 4.2: Переход на WebSocket-соединение с биржей**
    *   *Что сделаем:* Заменим клиентские REST-запросы на непрерывное WebSocket-подключение для мгновенной обработки тиков цены.
*   **📍 Шаг 4.3: Алгоритм Stealth Stop-Loss**
    *   *Что сделаем:* Запрограммируем хранение стоп-лоссов исключительно в оперативной памяти (RAM) сервера во Франкфурте (без отправки на публичную биржу). При касании триггера сервер отправляет моментальный рыночный ордер, защищая сделку от банковских HFT-охотников за стопами.
*   **📍 Шаг 4.4: Серверный EOD Auto-Halt Cron**
    *   *Что сделаем:* Настроим автоматический Cron-скрипт на сервере, который ровно в 18:00 CET закроет все сделки и остановит автопилот, даже если у пользователя выключен компьютер или разрядился телефон.

## 🧪 АКАДЕМИЧЕСКАЯ ВЕРИФИКАЦИЯ (КРИТЕРИИ DOD - DEFINITION OF DONE)

Для демонстрации проекта профессору каждый спринт считается завершенным только при прохождении следующих тестов:
1.  **Тест RLS-безопасности:** Попытка отправить HTTP-запрос к API с неверным JWT-токеном должна возвращать статус `401 Unauthorized`.
2.  **Тест Circuit Breaker:** Искусственное закрытие демо-сделки с убытком в 50 EUR. Результат: Терминал мгновенно окрашивается в красный цвет, статус меняется на `PANIC STOP ACTIVE`, кнопки открытия сделок аппаратно блокируются.
3.  **Тест EOD Auto-Halt:** Перевод системного времени на 18:00 CET. Результат: Все открытые позиции мгновенно ликвидируются, прибыль фиксируется, статус системы меняется на `EOD PROTECTION SLEEP`.
