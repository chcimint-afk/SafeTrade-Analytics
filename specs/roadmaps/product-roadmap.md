# 🗺️ ТЕХНИЧЕСКАЯ ДОРОЖНАЯ КАРТА ПРОДУКТА (PRODUCT ROADMAP)
## Системное развитие функционала терминала SafeTrade Analytics v9
### Статус документа: Архитектурный план развития фич софта (Без финансовых метрик)

Этот документ представляет собой пошаговую техническую дорожную карту развития программного продукта **SafeTrade Analytics v9**. Он описывает последовательное проектирование, разработку и масштабирование технологического функционала системы от локального MVP до глобального HFT-терминала.

## 📐 ФАЗА 1: MVP И ВЕРСТКА ИНТЕРФЕЙСА (РЕЛИЗ 1.0 — ВЫПОЛНЕНО)
*Основной фокус: Создание визуального каркаса и базовой математики торгового риска в браузере.*

*   **Компонент 1.1: Bloomberg-style Dashboard (React/Tailwind)**
    *   *Функционал:* Темно-угольная графическая сетка, интерактивные макро-карты, лента Whale Activity и боковая панель статистики.
*   **Компонент 1.2: Client-side Simulator Engine**
    *   *Функционал:* Локальная симуляция движения котировок, расчет оптимального размера лота под риск 1% от баланса.
*   **Компонент 1.3: EOD Auto-Halt (Локальный таймер)**
    *   *Функционал:* Системная блокировка торговых кнопок на клиенте в 18:00 CET для фиксации дневного результата.

---

## 🔌 ФАЗА 2: БЭКЕНД И ОБЛАЧНАЯ СИНХРОНИЗАЦИЯ (РЕЛИЗ 2.0 — АКТИВНЫЙ ЭТАП)
*Основной фокус: Перенос локального состояния в защищенное облачное хранилище, авторизация и API-маршрутизация.*

*   **Компонент 2.1: Supabase PostgreSQL Database**
    *   *Функционал:* Развертывание реляционных таблиц `users` (балансы), `trades` (архив сделок) и `system_logs` (системные события).
*   **Компонент 2.2: Row Level Security (RLS) & JWT**
    *   *Функционал:* Безопасность на уровне базы данных. Внедрение SQL-политик `auth.uid() = user_id` для предотвращения утечки личных данных пользователей.
*   **Компонент 2.3: Next.js API Routes (Шлюзы)**
    *   *Функционал:* API-шлюзы для записи сделок (`/api/trading/record-trade`) и серверного контроля Circuit Breaker (`/api/trading/circuit-breaker`).
*   **Компонент 2.4: React Fetch Client**
    *   *Функционал:* Полное переключение интерфейса с localStorage на динамические запросы к бэкенду.

---

## ⚡ ФАЗА 3: УЗЕЛ С НИЗКИМ ПИНГОМ И WEBSOCKETS (РЕЛИЗ 3.0 — СЛЕДУЮЩИЙ ЭТАП)
*Основной фокус: Снижение сетевых задержек до физического минимума и скрытное исполнение ордеров.*

*   **Компонент 3.1: Cloud VPS Node (Франкфурт)**
    *   *Функционал:* Развертывание исполняемого Node.js ядра в непосредственной близости к серверам брокеров для минимизации пинга.
*   **Компонент 3.2: WebSocket Real-Time Stream**
    *   *Функционал:* Отказ от медленных HTTP REST-запросов. Перевод терминала на WebSocket-подключение для получения тиков цены за **5–15 миллисекунд**.
*   **Компонент 3.3: Stealth Stop-Loss Engine**
    *   *Функционал:* Разработка серверного алгоритма отслеживания стоп-лоссов в оперативной памяти (RAM) Франкфуртского узла. Ордера остаются невидимыми для внешних стаканов котировок, исключая «охоту за стопами».
*   **Компонент 3.4: Server-Side EOD Auto-Halt Cron**
    *   *Функционал:* Автономный скрипт на сервере, принудительно закрывающий сделки в 18:00 CET, независимо от статуса браузера пользователя.

---

## 🌐 ФАЗА 4: МУЛЬТИ-БРОКЕРСКИЕ АДАПТЕРЫ (РЕЛИЗ 4.0 — В ПЛАНАХ)
*Основной фокус: Масштабирование софта на внешние ликвидные рынки и адаптация под мобильные ОС.*

*   **Компонент 4.1: Unified Broker API Adaptors**
    *   *Функционал:* Разработка универсальных коннекторов-адаптеров к API крупнейших брокеров (Interactive Brokers API, Kraken API, Binance API) для автоматической сквозной торговли.
*   **Компонент 4.2: PWA (Progressive Web Application)**
    *   *Функционал:* Адаптация Next.js кода в прогрессивное веб-приложение для нативной, полноэкранной работы на смартфонах iOS и Android без скачивания из AppStore/GooglePlay.
*   **Компонент 4.3: ML Sentiment Aggregator**
    *   *Функционал:* Встраивание легковесного микросервиса на Python для сбора и экспресс-анализа новостных лент Reuters/Bloomberg с автоматическим расчетом индекса паники в Threat Detector.
*   **Компонент 4.4: Multi-Tenant Sub-Account Dashboard**
    *   *Функционал:* Добавление в интерфейс возможности мгновенного переключения между тремя независимыми суб-аккаунтами брокеров с единого терминала для диверсификации торговых лимитов.
