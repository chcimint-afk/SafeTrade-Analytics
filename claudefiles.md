ВНИМАНИЕ ВСЕМ ИИ-АГЕНТАМ И МОДЕЛЯМ (CLAUDE, GPT И ДР.)!
Этот проект разрабатывается по строгим правилам безопасности и качества. Любые изменения в коде должны проходить через автоматические проверки.

Перед началом работы и перед коммитом изменений вы ОБЯЗАНЫ выполнить следующие требования:

------------------------------------------------------------------------
1. МЕТОДОЛОГИЯ ПРОЕКТИРОВАНИЯ (SUPERPOWER & MEMORY)
------------------------------------------------------------------------
* Сначала планируйте: Никогда не пишите код сразу. Сначала составьте план изменений в папке `plan/` и согласуйте с пользователем.
* Пользуйтесь памятью: Сверяйтесь с техническим ТЗ в папке `specs/`.

------------------------------------------------------------------------
2. ТРОЙНОЙ АУДИТ КОДА (INSTAGRAM-МЕТОД)
------------------------------------------------------------------------
Перед сдачей задачи запустите три обязательные проверки безопасности в папке `app-code/`:
1. Сканирование логики и синтаксиса (ESLint):
   👉 `npm run lint` (В коде должно быть 0 ошибок).
2. Проверка типов TypeScript (tsc):
   👉 `npx tsc --noEmit` (Должно быть 0 ошибок типов).
3. Проверка безопасности внешних пакетов (NPM Audit):
   👉 `npm audit` (Должно быть 0 уязвимостей).

------------------------------------------------------------------------
3. ЭМУЛЯЦИЯ КОМАНДЫ РАЗРАБОТКИ (G-STACK / SUBAGENTS)
------------------------------------------------------------------------
* Не отправляйте код без рецензии.
* Создайте виртуального субагента-тестировщика (QA) и дайте ему задачу:
  "Внимательно изучи мой код, найди потенциальные баги, проверь обработку ошибок (catch blocks) и попытайся сломать логику".
* Сохраняйте код только после того, как субагент подтвердит, что багов нет.

------------------------------------------------------------------------
4. ГЛОБАЛЬНЫЕ НАСТРОЙКИ БЕЗОПАСНОСТИ
------------------------------------------------------------------------
* Секреты: Никогда не пишите пароли и API-ключи текстом в коде. Все секреты должны быть только в файле `.env.local` или в настройках окружения Vercel.
* Сетевая защита: В файле `next.config.ts` должны быть активны HTTP-заголовки безопасности (X-Frame-Options: DENY, nosniff, Referrer-Policy, Permissions-Policy).
